# DR CLI Telemetry Configuration

This `.dr-config.yaml` file configures OpenTelemetry export for the `dr` CLI tool.

## Location

- **Development**: Copy to `~/.dr-config.yaml` in your home directory
- **Container**: Already mounted at `~/.dr-config.yaml` in the rounds container

## Configuration

```yaml
telemetry:
  otlp:
    endpoint: "http://192.168.0.117:4318/v1/traces"
    logs_endpoint: "http://192.168.0.117:4318/v1/logs"
    service_name: "dr-cli"
```

## Testing

Verify telemetry is working:

```bash
# Check telemetry status
dr version

# Should show:
# Telemetry: enabled, configured

# Run with debug output
DR_TELEMETRY_DEBUG=1 dr version

# Should show:
# [TELEMETRY] Export SUCCESS - 2 span(s) sent
```

down from ~13ms before optimizations.
